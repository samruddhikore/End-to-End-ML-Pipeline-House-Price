6.4 Option 2: Flask REST API

Building a Flask API

Implementation Strategy

Create api.py for REST endpoint:

from flask import Flask, request, jsonify
import pickle
import numpy as np

app = Flask(__name__)

# Load model once at startup
with open('models/model.pkl', 'rb') as f:
    model = pickle.load(f)

@app.route('/predict', methods=['POST'])
def predict():
    data = request.get_json()
    
    # Extract features
    features = np.array([[
        data['sqft'],
        data['bedrooms'],
        data['garden']
    ]])
    
    # Make prediction
    prediction = model.predict(features)[0]
    
    return jsonify({
        'predicted_price': round(prediction, 2),
        'inputs': data
    })

if __name__ == '__main__':
    app.run(debug=False, host='0.0.0.0', port=5000)

API Usage Example

Request

POST /predict
Content-Type: application/json

{
    "sqft": 2000,
    "bedrooms": 4,
    "garden": 1
}

Response

{
    "predicted_price": 314539.17,
    "inputs": {
        "sqft": 2000,
        "bedrooms": 4,
        "garden": 1
    }
}

6.5 Option 3: Docker Containerization

Docker Setup

Dockerfile

FROM python:3.9

WORKDIR /app

COPY requirements.txt .
RUN pip install -r requirements.txt

COPY . .

EXPOSE 5000

CMD ["python", "api.py"]

Building & Running

# Build image
docker build -t house-price-predictor .

# Run container
docker run -p 5000:5000 house-price-predictor

Docker Benefits

  • Consistent environment across development and production
  • Easy deployment to any cloud platform (AWS, GCP, Azure)
  • Horizontal scaling with container orchestration (Kubernetes)
  • Automatic dependency management

6.6 Option 4: Cloud Deployment

  1. Create Lambda function from Docker image or ZIP

6.8 Model Monitoring & Maintenance

Performance Monitoring

Key Metrics to Track

  • Prediction Accuracy: Compare predictions vs actual sold prices
  • Error Distribution: Monitor for systematic biases
  • Data Drift: Market changes affecting feature distributions
  • Prediction Latency: API response time performance
  • System Reliability: Uptime and error rates

Retraining Strategy

When to Retrain

  • Quarterly: Regular scheduled retraining with new data
  • On Demand: If prediction error degrades by > 5%
  • After Major Events: Market crashes, economic shifts
  • Data Collection: Accumulate 100+ new samples before retraining

Monitoring Setup

6.10 API Security & Authentication

Security Measures

For Internal/Private APIs

  • API key authentication (simple, for internal use)
  • HTTPS/TLS encryption for all communications
  • Rate limiting to prevent abuse
  • Input validation on all parameters

For Public APIs

  • OAuth 2.0 authentication
  • JWT tokens for stateless authentication
  • CORS policy restrictions
  • WAF (Web Application Firewall)
  • DDoS protection
