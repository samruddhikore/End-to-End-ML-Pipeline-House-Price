2.5 Feature Engineering & Selection

Feature Selection Strategy

Given the small feature set (3 input features), all features were retained for the initial model. Feature selection was guided by:

  • Correlation with Target: All features show meaningful correlation with price
  • Domain Knowledge: All features are known to influence house prices
  • Practical Relevance: All features are easily measurable at time of prediction
  • Model Performance: Including all features optimizes model accuracy

Feature Importance Analysis

Feature Correlation with Price Model Coefficient Relative Importance
sqft 0.988 (Very Strong) $139.16 per sqft Strongest predictor
bedrooms 0.970 (Very Strong) $14,363.47 per bedroom Extremely important
garden 0.187 (Weak-Moderate) $13,629.44 for garden Moderate importance

Feature Scaling Decision

Decision: No feature scaling applied

Reasoning: Linear regression is scale-invariant in terms of predictions. Features are already on similar magnitude scales (thousands to tens of thousands). Interpretability is improved without scaling.

2.6 Model Selection & Justification

Why Linear Regression?

✓ Advantages

  • Simple and interpretable - coefficients directly show feature impact
  • Fast to train and make predictions
  • Features show strong linear relationships with target (r > 0.95)
  • Small dataset (65 samples) - simpler models generalize better
  • No hyperparameter tuning needed - reduces complexity
  • Statistical properties well-understood and documented
  • Excellent baseline against which to compare other models

Considered Alternatives

Model Type Why Not Selected When to Use Instead
Polynomial Regression Risk of overfitting; small dataset makes high-degree polynomials dangerous If data shows non-linear relationships; with larger datasets
Random Forest Overkill for simple problem; harder to interpret; needs tuning With many features or categorical variables; non-linear patterns
Neural Networks Requires much more data; black-box nature; expensive computation With very large datasets and complex patterns
SVM Hard to interpret; hyperparameter tuning required; small benefit With high-dimensional data or non-linear boundaries

2.7 Model Training Process

Data Splitting Strategy

Train-Test Split

Method: 80% training, 20% testing
Training Sets: 52 samples
Testing Set: 13 samples
Random State: 42 (for reproducibility)
Stratification: Not applied (continuous target, no stratification needed)

Model Training

Training Process

The linear regression model fits using the Ordinary Least Squares (OLS) method:

minimize: Σ(y_i - ŷ_i)²

where y_i = actual price, ŷ_i = predicted price

The model learns coefficients that minimize the sum of squared residuals on training data.

Resulting Model Equation

Price Prediction Formula

Price = -34,864.15 + (139.16 × sqft) + (14,363.47 × bedrooms) + (13,629.44 × garden)

Interpretation

  • Intercept (-$34,864.15): Base value before adding feature contributions (theoretical baseline)
  • sqft coefficient ($139.16): Each additional sq ft adds this amount to price
  • bedrooms coefficient ($14,363.47): Each additional bedroom adds this amount
  • garden coefficient ($13,629.44): Having a garden adds this fixed amount

2.8 Model Evaluation Strategy

Evaluation Metrics

Metric Formula Interpretation Ideal Value
R² Score 1 - (SS_res / SS_tot) % of variance explained (0-1 range) 1.0 (closer to 1 is better)
RMSE √(Σ(y_i - ŷ_i)² / n) Average magnitude of errors (same units as target) Smaller is better
MAE Σ|y_i - ŷ_i| / n Mean absolute error (more robust than RMSE) Smaller is better
Residuals y_i - ŷ_i Prediction errors for analysis Normally distributed, centered at 0

Cross-Validation Approach

Not Applied in Baseline: Given small dataset (65 samples), simple train-test split used.

Recommendation: For enhanced validation, use 5-fold cross-validation:

  • Divides data into 5 equal folds
  • Trains 5 models, each using 4 folds for training, 1 for validation
  • Provides more robust performance estimate
  • Better utilizes small dataset

2.9 Hyperparameter Tuning

Linear regression has no hyperparameters to tune.

The model is fully determined by:

  • The training data (X, y)
  • The OLS optimization criterion
  • The choice of features included

Therefore, all efforts in model optimization focused on data quality and feature engineering rather than hyperparameter tuning.
